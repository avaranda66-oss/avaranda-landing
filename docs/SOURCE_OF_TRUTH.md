# Fonte Da Verdade

Este projeto deve ser guiado por uma hierarquia clara. Quando uma AI, pessoa ou ferramenta fizer alteracoes, a ordem de decisao e:

1. `DESIGN.md`: tokens, tom visual, restricoes e componentes.
2. `mockupexample/`: referencia visual de fidelidade desktop e mobile.
3. `images/`: assets oficiais disponiveis no projeto.
4. `docs/PROJECT_STATUS.md`: o que ja foi feito e o que falta.
5. Codigo em `src/`: implementacao atual, sempre subordinada ao `DESIGN.md`.

## Regra principal

Se uma decisao visual mudar, atualizar primeiro o `DESIGN.md`. Depois ajustar o codigo.

## Checkpoint antes de cada ciclo

- O primeiro viewport mostra marca, historia de Tarcila e reserva?
- O fundo ainda parece papel antigo?
- A tipografia ainda parece editorial e afetiva?
- Fotos de comida estao apetitosas e legiveis?
- Mobile tem acordeao de capitulos e CTA claro?
